## 28.1 无监督学习回顾

前面的回归和分类都是**监督学习**——数据带标签（正确答案）。而**无监督学习**的数据**没有标签**，目标是让模型自己发现数据内在的结构。两大典型任务：

- **聚类**：把相似的样本自动分成若干组。例如电商给用户分群、把新闻按主题归类。
- **降维**：在尽量保留信息的前提下减少特征数量。用于可视化、压缩、去噪、加速训练。

---

## 28.2 K-Means 聚类

**K-Means** 是最常用的聚类算法。思想很直观：

1. 你指定要分成 **K** 个簇。
2. 算法随机放 K 个“簇中心”。
3. 每个样本归到离它最近的中心，形成 K 个簇。
4. 把每个中心移动到本簇所有点的平均位置。
5. 重复 3~4，直到中心不再移动。

### 准备数据并聚类

我们先用 `make_blobs` 生成几团明显的数据来演示：

```python
import numpy as np
from sklearn.datasets import make_blobs
from sklearn.cluster import KMeans

# 生成 4 团数据（我们假装不知道是 4 团）
X, _ = make_blobs(n_samples=300, centers=4, cluster_std=0.8, random_state=42)

kmeans = KMeans(n_clusters=4, random_state=42, n_init=10)
labels = kmeans.fit_predict(X)      # 训练并返回每个样本的簇编号

print("前 10 个样本的簇：", labels[:10])
print("簇中心：\n", kmeans.cluster_centers_.round(2))
print("Inertia（簇内误差）：", round(kmeans.inertia_, 1))
```

- `fit_predict` 一步完成训练并返回聚类标签（0~K-1）。
- `cluster_centers_` 是每个簇的中心坐标。
- `inertia_` 是“簇内平方和”——所有点到各自簇中心距离的平方和，**越小说明簇越紧凑**。
- `n_init=10`：用 10 组不同的初始中心各跑一次取最好的，避免陷入差的局部解。

### 可视化聚类结果

```python
import matplotlib.pyplot as plt
plt.rcParams["font.sans-serif"] = ["SimHei"]
plt.rcParams["axes.unicode_minus"] = False

plt.scatter(X[:, 0], X[:, 1], c=labels, cmap="viridis", alpha=0.6)
plt.scatter(kmeans.cluster_centers_[:, 0], kmeans.cluster_centers_[:, 1],
            c="red", marker="X", s=200, label="簇中心")
plt.title("K-Means 聚类结果")
plt.legend()
plt.show()
```

`c=labels` 让不同簇显示成不同颜色，红色大叉是簇中心。

## 28.3 如何选择 K：肘部法则

现实中我们并不知道该分几类。**肘部法则（Elbow Method）**帮我们选：尝试不同的 K，画出 K 与 inertia 的关系曲线。inertia 会随 K 增大而减小，但在“合适的 K”之后下降会明显变缓，曲线出现一个“肘部”拐点。

```python
inertias = []
K_range = range(1, 8)
for k in K_range:
    km = KMeans(n_clusters=k, random_state=42, n_init=10)
    km.fit(X)
    inertias.append(km.inertia_)

plt.plot(K_range, inertias, marker="o")
plt.xlabel("K（簇数）")
plt.ylabel("Inertia")
plt.title("肘部法则")
plt.show()

print(dict(zip(K_range, [round(i) for i in inertias])))
```

对本例数据，inertia 从 K=1 到 K=4 急剧下降，之后趋于平缓——“肘部”出现在 **K=4**，这正是我们生成数据时的真实簇数。

## 28.4 轮廓系数：评估聚类好坏

聚类没有真实标签可对照，怎么评估质量？**轮廓系数（Silhouette Score）**衡量“簇内紧密、簇间分离”的程度，取值 -1~1，**越接近 1 越好**：

```python
from sklearn.metrics import silhouette_score

score = silhouette_score(X, labels)
print(f"轮廓系数：{score:.3f}")      # 本例约 0.83，说明聚类效果很好
```

可以用它辅助选 K：对不同 K 计算轮廓系数，选最高的那个。

## 28.5 降维与 PCA

当数据有很多特征（几十上百维）时，我们无法直接可视化，计算也慢。**降维**能把高维数据压缩到低维，同时尽量保留原有信息。

**主成分分析（PCA, Principal Component Analysis）**是最经典的降维方法。它找到数据中方差最大的几个方向（“主成分”），把数据投影上去。

> PCA 对特征量纲敏感，使用前通常要先**标准化**。

```python
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

iris = load_iris()
X_scaled = StandardScaler().fit_transform(iris.data)   # 4 个特征，先标准化

pca = PCA(n_components=2)          # 降到 2 维
X_pca = pca.fit_transform(X_scaled)

print("降维前：", X_scaled.shape)   # (150, 4)
print("降维后：", X_pca.shape)      # (150, 2)
print("各主成分解释的方差比例：", pca.explained_variance_ratio_.round(3))
print("累计解释比例：", round(pca.explained_variance_ratio_.sum(), 3))
# 约 0.958，即 2 个主成分保留了原始 4 维约 96% 的信息
```

`explained_variance_ratio_` 告诉我们每个主成分保留了多少信息。这里用 2 维就保留了约 96%，非常划算——既能画图，又几乎不丢信息。

### 用 PCA 可视化

把 4 维鸢尾花降到 2 维后就能画在平面上，用真实品种上色，可以看到三个品种分得很开：

```python
import matplotlib.pyplot as plt

plt.figure(figsize=(7, 5))
scatter = plt.scatter(X_pca[:, 0], X_pca[:, 1], c=iris.target, cmap="viridis")
plt.xlabel("主成分 1")
plt.ylabel("主成分 2")
plt.title("鸢尾花数据 PCA 降维可视化")
plt.colorbar(scatter, label="品种")
plt.show()
```

## 28.6 综合：先降维，再聚类

无监督方法可以组合使用。比如对鸢尾花（假装没有标签），先聚类，再和真实品种对比：

```python
from sklearn.cluster import KMeans
from sklearn.metrics import adjusted_rand_score

kmeans = KMeans(n_clusters=3, random_state=42, n_init=10)
cluster_labels = kmeans.fit_predict(X_scaled)

# 调整兰德指数：衡量聚类结果与真实标签的一致性（1 表示完全一致）
ari = adjusted_rand_score(iris.target, cluster_labels)
print(f"聚类与真实品种的一致性 (ARI)：{ari:.3f}")   # 约 0.62
```

ARI 约 0.62，说明在完全没有标签的情况下，K-Means 也能把鸢尾花大致分成对应三个品种的簇（其中两个品种本身就比较像，不易完全分开）。这展示了无监督学习的价值——从无标签数据中发现真实存在的结构。

---

## 本章小结

- 无监督学习无标签，发现数据结构：**聚类**（分组）、**降维**（压缩特征）。
- **K-Means**：指定 K，迭代地“分配-更新中心”；`fit_predict` 得簇标签，`inertia_` 看紧凑度，`n_init=10` 更稳。
- 选 K：**肘部法则**（inertia 曲线拐点）；评估：**轮廓系数**（越接近 1 越好）。
- **PCA** 降维：保留方差最大的方向；用前先标准化；`explained_variance_ratio_` 看保留信息量。
- 降到 2 维可用于可视化；无监督方法可组合（如降维 + 聚类）。

X, _ = make_blobs(n_samples=200, centers=3, cluster_std=1.0, random_state=0)

inertias = [KMeans(n_clusters=k, random_state=42, n_init=10).fit(X).inertia_ for k in range(1, 8)]

**第 3 题**：

```python
from sklearn.metrics import silhouette_score
for k in [2, 3, 4, 5]:
    lb = KMeans(n_clusters=k, random_state=42, n_init=10).fit_predict(X)
    print(f"K={k}: {silhouette_score(X, lb):.3f}")
# K=3 通常最高
```

**第 4 题**：

```python
from sklearn.datasets import load_wine
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

wine = load_wine()
Xs = StandardScaler().fit_transform(wine.data)
X2 = PCA(n_components=2).fit_transform(Xs)
pca = PCA(n_components=2).fit(Xs)
print("保留信息：", round(pca.explained_variance_ratio_.sum(), 3))   # 约 0.55

plt.scatter(X2[:, 0], X2[:, 1], c=wine.target, cmap="viridis")
plt.title("红酒数据 PCA 降维")
plt.show()
```

**第 5 题**：当特征极多（维度高）、存在冗余/相关特征、或需要可视化、加速训练、缓解“维度灾难”时，可先降维再建模。代价：降维会**丢失部分信息**，且 PCA 得到的新特征是原特征的线性组合，**失去了原本的业务可解释性**（“主成分1”不再对应某个具体指标）。
