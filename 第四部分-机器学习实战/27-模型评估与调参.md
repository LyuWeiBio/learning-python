## 27.1 过拟合与欠拟合

模型的目标是**举一反三**——在训练时没见过的新数据上也表现良好（称为**泛化能力**）。但常出现两种毛病：

- **欠拟合（Underfitting）**：模型太简单，连训练数据的规律都没学好。表现：训练集和测试集都差。
- **过拟合（Overfitting）**：模型太复杂，把训练数据背得死死的（连噪声都记住了），一到新数据就露馅。表现：训练集很好，测试集很差。

```
欠拟合            刚刚好            过拟合
训练差 测试差   训练好 测试好    训练极好 测试差
（太笨）        （理想）         （死记硬背）
```

**判断方法**：同时看训练集和测试集得分。

```python
print("训练集得分：", model.score(X_train, y_train))
print("测试集得分：", model.score(X_test, y_test))
```

- 两者都低 → 欠拟合（换更强的模型、加特征）。
- 训练高、测试低，差距大 → 过拟合（简化模型、加正则、加数据、限制树深等）。

第 26 章限制决策树 `max_depth` 正是为了防止过拟合。

## 27.2 为什么需要交叉验证

我们一直用“一次”训练集/测试集划分来评估。但这个结果可能**碰运气**——万一划分时正好把简单样本分到了测试集，得分就虚高。

**交叉验证（Cross-Validation）**让评估更可靠。最常用的 **K 折交叉验证**做法：

1. 把训练数据平均分成 K 份（比如 5 份）。
2. 轮流用其中 1 份当验证集、其余 4 份训练，共训练 K 次。
3. 得到 K 个得分，取平均作为最终评估。

这样每个样本都当过一次验证集，结果更稳定、更可信。

```
K=5 折：
第1轮  [验证][训练][训练][训练][训练]
第2轮  [训练][验证][训练][训练][训练]
...      每轮换一份做验证，最后平均 5 个得分
```

## 27.3 用 cross_val_score 做交叉验证

```python
from sklearn.datasets import load_wine
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.preprocessing import StandardScaler
from sklearn.pipeline import Pipeline
from sklearn.neighbors import KNeighborsClassifier
import numpy as np

data = load_wine()
X_train, X_test, y_train, y_test = train_test_split(
    data.data, data.target, test_size=0.2, random_state=42, stratify=data.target)

# 用 Pipeline 把标准化和模型绑在一起（交叉验证时每折都会正确地只在训练部分做标准化）
pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("knn", KNeighborsClassifier()),
])

scores = cross_val_score(pipe, X_train, y_train, cv=5)   # 5 折
print("每折得分：", scores.round(3))
print(f"平均得分：{scores.mean():.3f} (±{scores.std():.3f})")
# 平均得分：约 0.965
```

> **为什么在这里必须用 Pipeline？** 交叉验证会反复划分训练数据。如果先对全部数据做标准化再交叉验证，验证折的信息会通过“均值/标准差”泄露到训练中。用 Pipeline，标准化会在每一折内部**只对该折的训练部分**拟合，杜绝泄露。这是专业做法。

## 27.4 参数 vs 超参数

- **参数（parameters）**：模型从数据中**自己学出来的**，如线性回归的系数、截距。我们不用管它们。
- **超参数（hyperparameters）**：**我们在训练前手动设定的**，模型学不出来，如 KNN 的 `n_neighbors`、决策树的 `max_depth`、逻辑回归的正则强度 `C`。

超参数的好坏对效果影响很大，**调参**就是寻找最优超参数组合的过程。手动一个个试太麻烦，我们用工具自动搜索。

---

## 27.5 网格搜索 GridSearchCV

`GridSearchCV` 会**遍历你给定的所有超参数组合**，对每个组合做交叉验证，自动选出最佳的那组。

```python
from sklearn.model_selection import GridSearchCV

# 要搜索的超参数网格
# 注意命名：Pipeline 里步骤名 + 双下划线 + 参数名
param_grid = {
    "knn__n_neighbors": [3, 5, 7, 9, 11],
    "knn__weights": ["uniform", "distance"],
}

grid = GridSearchCV(pipe, param_grid, cv=5, scoring="accuracy")
grid.fit(X_train, y_train)

print("最佳超参数：", grid.best_params_)
print(f"最佳交叉验证得分：{grid.best_score_:.3f}")
print(f"在测试集上的得分：{grid.score(X_test, y_test):.3f}")
```

输出类似（具体结果因数据划分略有不同）：

```
最佳超参数： {'knn__n_neighbors': 11, 'knn__weights': 'uniform'}
最佳交叉验证得分：0.972
在测试集上的得分：1.000
```

- `grid.best_params_`：找到的最佳超参数组合。
- `grid.best_score_`：该组合的交叉验证平均得分。
- `grid.best_estimator_`：用最佳超参数、在全部训练集上重新训练好的模型，可直接拿去 `predict`。

网格会把所有组合都试一遍（这里 5×2=10 组，每组 5 折 = 50 次训练），组合太多会很慢。

---

## 27.6 随机搜索（了解）

当超参数很多、取值范围很大时，遍历所有组合代价太高。`RandomizedSearchCV` 随机采样一部分组合来试，用更少的时间找到不错的结果：

```python
from sklearn.model_selection import RandomizedSearchCV

param_dist = {"knn__n_neighbors": list(range(1, 30))}
rand = RandomizedSearchCV(pipe, param_dist, n_iter=10, cv=5, random_state=42)
rand.fit(X_train, y_train)
print(rand.best_params_)
```

经验：超参数组合少用 `GridSearchCV`（穷尽），组合多用 `RandomizedSearchCV`（高效）。

---

## 27.7 规范的建模流程

把本部分学到的整合起来，一个专业的建模流程是：

1. **划分**训练集 / 测试集（测试集"封存"，只在最后用一次）。
2. 用 **Pipeline** 组合预处理和模型。
3. 用 **交叉验证 + GridSearchCV** 在训练集上调参、选模型。
4. 用 `best_estimator_` 在**测试集**上做最终评估（这是对泛化能力的诚实估计）。
5. 满意后，用**全部数据**重新训练，部署上线。

> **黄金原则**：测试集只能在最后用一次。如果你反复用测试集调参，就等于把测试集也“学”进去了，评估会虚高。调参要用交叉验证（或单独的验证集）。

---

## 本章小结

- 过拟合（训练好测试差）vs 欠拟合（都差）；同时看训练/测试得分来判断。
- 一次划分可能碰运气；**K 折交叉验证**更可靠：轮流验证、取平均。
- `cross_val_score(pipe, X, y, cv=5)`；预处理务必放进 **Pipeline** 防泄露。
- 参数（模型自学）vs 超参数（人工设定，需调）。
- `GridSearchCV` 遍历超参数组合 + 交叉验证选最优；`best_params_/best_score_/best_estimator_`。
- 组合多时用 `RandomizedSearchCV`；**测试集只在最后用一次**。

## 练习题

1. **诊断过拟合**：训练一棵 `max_depth=None`（不限深度）的决策树，分别打印训练集和测试集得分，判断是否过拟合。
2. **交叉验证**：对逻辑回归（配 StandardScaler 的 Pipeline）在红酒数据上做 10 折交叉验证，输出平均得分。
3. **网格搜索决策树**：用 `GridSearchCV` 搜索决策树的最佳 `max_depth`（在 1~10 中），输出最佳深度与得分。
4. **多参数搜索**：对随机森林搜索 `n_estimators`（[50,100,200]）和 `max_depth`（[3,5,None]）的最佳组合。
5. **思考题**：为什么调参必须用交叉验证或验证集，而不能直接用测试集反复试？

**第 1 题**：

```python
from sklearn.tree import DecisionTreeClassifier
t = DecisionTreeClassifier(random_state=42).fit(X_train, y_train)
print("训练集：", t.score(X_train, y_train))   # 1.0（完美记住训练集）
```

**第 2 题**：

```python
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.model_selection import cross_val_score

pipe_lr = make_pipeline(StandardScaler(), LogisticRegression(max_iter=5000))
scores = cross_val_score(pipe_lr, X_train, y_train, cv=10)
print(f"平均得分：{scores.mean():.3f}")   # 约 0.98
```

**第 3 题**：

```python
from sklearn.model_selection import GridSearchCV
from sklearn.tree import DecisionTreeClassifier

grid = GridSearchCV(
    DecisionTreeClassifier(random_state=42),
    {"max_depth": list(range(1, 11))}, cv=5)
grid.fit(X_train, y_train)
print(grid.best_params_, round(grid.best_score_, 3))
```

**第 4 题**：

```python
from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import GridSearchCV

grid = GridSearchCV(
    RandomForestClassifier(random_state=42),
    {"n_estimators": [50, 100, 200], "max_depth": [3, 5, None]}, cv=5)
grid.fit(X_train, y_train)
print("最佳组合：", grid.best_params_)
print(f"测试集得分：{grid.score(X_test, y_test):.3f}")
```

**第 5 题**：测试集是用来**诚实估计模型在未知数据上表现**的。如果反复用它调参，模型的超参数就会逐渐“迎合”测试集，相当于把测试集信息泄露进了模型，最终得分会偏乐观、不能反映真实泛化能力。所以调参必须在训练集内部用交叉验证/验证集完成，测试集留到最后只评估一次。
