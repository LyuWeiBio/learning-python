## 23.1 什么是 EDA

**探索性数据分析（Exploratory Data Analysis, EDA）**是拿到数据后、建模之前的关键步骤：通过统计和可视化，全面“了解”数据——它长什么样、有没有问题、变量之间有什么关系。EDA 做得好，后面的机器学习才有好地基。

一个通用的 EDA 流程：

1. **加载数据**，看整体规模与字段。
2. **检查质量**：缺失值、重复值、异常值、数据类型。
3. **单变量分析**：每个变量各自的分布。
4. **多变量分析**：变量之间的关系。
5. **分组洞察**：按类别汇总，发现规律。
6. **总结结论**。

## 23.2 准备数据

为了让你能完整复现，我们用代码**生成**一份模拟的电商销售数据（真实项目中通常是 `pd.read_csv("xxx.csv")` 读进来的）：

```python
import numpy as np
import pandas as pd

np.random.seed(42)          # 固定随机种子，保证每次结果一致
n = 500

df = pd.DataFrame({
    "订单ID": range(1, n + 1),
    "类别": np.random.choice(["电子产品", "服装", "食品", "家居", "图书"], n),
    "地区": np.random.choice(["华北", "华东", "华南", "西部"], n, p=[0.3, 0.3, 0.25, 0.15]),
    "销售额": np.round(np.random.gamma(2, 300, n), 2),
    "数量": np.random.randint(1, 11, n),
    "客户年龄": np.random.randint(18, 65, n).astype(float),
})
df["利润"] = np.round(df["销售额"] * np.random.uniform(0.05, 0.3, n), 2)

# 人为制造一些缺失值，模拟真实的"脏数据"
missing_idx = np.random.choice(n, 25, replace=False)
df.loc[missing_idx, "客户年龄"] = np.nan
```

## 23.3 第一步：整体认识数据

```python
print(df.shape)          # (500, 7) 有多少行多少列
print(df.head())         # 看前几行长啥样
print(df.info())         # 字段、类型、非空数量
print(df.describe())     # 数值列的统计摘要
```

从 `info()` 我们能看到“客户年龄”有缺失（非空数量 < 500），从 `describe()` 能看到销售额、利润、年龄的大致范围和均值。

---

## 23.4 第二步：检查数据质量

```python
# 缺失值
print("各列缺失值：")
print(df.isnull().sum())

# 重复值
print("重复行数：", df.duplicated().sum())

# 处理缺失：用中位数填充年龄（比均值更抗异常值）
df["客户年龄"] = df["客户年龄"].fillna(df["客户年龄"].median())
print("填充后缺失：", df["客户年龄"].isnull().sum())
```

> 用**中位数**而非均值填充，是因为中位数不容易被极端值带偏，在有异常值时更稳健。

---

## 23.5 第三步：单变量分析

### 数值变量的分布

```python
import matplotlib.pyplot as plt
plt.rcParams["font.sans-serif"] = ["SimHei"]     # 中文显示，按系统调整
plt.rcParams["axes.unicode_minus"] = False

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
axes[0].hist(df["销售额"], bins=30, edgecolor="black")
axes[0].set_title("销售额分布")
axes[1].hist(df["客户年龄"], bins=20, edgecolor="black", color="orange")
axes[1].set_title("客户年龄分布")
plt.tight_layout()
plt.show()
```

观察销售额直方图，通常会发现它**右偏**（大部分订单金额较小，少数大额订单拉长了右尾）——这在真实销售数据中很常见。

### 类别变量的分布

```python
print(df["类别"].value_counts())
print(df["地区"].value_counts(normalize=True))   # 各地区占比

df["类别"].value_counts().plot(kind="bar", title="各类别订单数")
plt.tight_layout()
plt.show()
```

---

## 23.6 第四步：多变量分析

### 数值变量间的相关性

```python
# 相关系数矩阵（只对数值列）
num_cols = ["销售额", "数量", "客户年龄", "利润"]
corr = df[num_cols].corr()
print(corr)
```

相关系数在 -1 到 1 之间：接近 1 表示强正相关，接近 -1 表示强负相关，接近 0 表示几乎无线性关系。这里“销售额”和“利润”应呈较强正相关（因为利润是按销售额比例算的）。

用散点图直观查看：

```python
plt.scatter(df["销售额"], df["利润"], alpha=0.5)
plt.xlabel("销售额")
plt.ylabel("利润")
plt.title("销售额 vs 利润")
plt.show()
```

---

## 23.7 第五步：分组洞察（业务价值所在）

这是最能产出“结论”的一步——用 `groupby` 从不同维度汇总：

```python
# 每个类别的总销售额和平均利润
category_stats = df.groupby("类别").agg(
    总销售额=("销售额", "sum"),
    平均利润=("利润", "mean"),
    订单数=("订单ID", "count"),
).sort_values("总销售额", ascending=False)
print(category_stats)

# 每个地区的总利润
region_profit = df.groupby("地区")["利润"].sum().sort_values(ascending=False)
print(region_profit)

# 可视化：各类别总销售额
category_stats["总销售额"].plot(kind="bar", title="各类别总销售额")
plt.tight_layout()
plt.show()
```

还可以做**交叉分析**（两个维度）：

```python
# 每个地区、每个类别的平均销售额（透视表）
pivot = df.pivot_table(values="销售额", index="地区", columns="类别", aggfunc="mean")
print(pivot.round(1))
```

`pivot_table` 是 Excel 数据透视表的 Python 版，做多维汇总非常强大。

---

## 23.8 第六步：分箱分析

把连续变量分段，能发现分段规律。比如按年龄段看消费：

```python
# 用 pd.cut 把年龄分箱
df["年龄段"] = pd.cut(df["客户年龄"],
                      bins=[0, 25, 35, 45, 100],
                      labels=["青年", "青壮年", "中年", "中老年"])

print(df.groupby("年龄段", observed=True)["销售额"].mean())
```

`pd.cut` 按区间把数值切成类别，配合 `groupby` 就能分析“不同年龄段的消费差异”。

---

## 23.9 总结分析结论

一份好的 EDA 最终要能用几句话概括发现。基于本章数据，我们可以总结（**具体数值取决于随机数据，你运行时看到的会略有不同**）：

- 数据共 500 条订单，客户年龄有约 5% 缺失，已用中位数填充。
- 销售额分布明显**右偏**（偏度 > 1）：多数订单集中在中低金额，存在少量大额订单。
- 销售额与利润呈**较强正相关**（相关系数约 0.8）——这符合“利润按销售额比例计算”的设定。
- 数量、客户年龄与销售额几乎不相关（相关系数接近 0）。
- 各类别的订单数与总销售额比较接近（因为类别是等概率随机分配的）；真实业务数据往往会有明显的“头部品类”。

> 在真实项目里，这些结论会直接指导后续决策：主推哪个品类、重点运营哪个地区、如何针对不同人群营销。这也正是机器学习建模前要想清楚的问题。**重点不是记住这些数字，而是掌握“提问—统计—可视化—下结论”的分析套路。**

## 本章小结

- EDA 六步：加载 → 查质量 → 单变量 → 多变量 → 分组洞察 → 结论。
- 认识数据：`shape/head/info/describe`。
- 质量检查：`isnull().sum()`、`duplicated()`；缺失可用中位数/均值填充。
- 单变量：数值看直方图，类别看 `value_counts` + 柱状图。
- 多变量：`corr()` 相关系数 + 散点图。
- 分组洞察：`groupby().agg()`、`pivot_table`；`pd.cut` 分箱。
- 目标是把数据“讲成一个故事”，得出可指导行动的结论。
