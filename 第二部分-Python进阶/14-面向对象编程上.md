## 14.1 什么是面向对象

之前我们用变量存数据、用函数处理数据，数据和操作是**分开**的。**面向对象编程（Object-Oriented Programming, OOP）**换了一种思路：把**数据**和**操作数据的方法**捆绑在一起，形成一个个“对象”。

打个比方：

- **类（class）** 是“模板 / 图纸”，比如“狗”这个概念。
- **对象（object / 实例 instance）** 是按图纸造出来的“具体实物”，比如你家那只叫“旺财”的狗。

一个类可以创建出无数个对象，每个对象有自己的数据，但共享同样的方法。生活中处处是对象：一个学生、一辆车、一个银行账户……OOP 让代码更贴近现实、更易组织。

其实我们一直在用对象：字符串是 `str` 类的对象，列表是 `list` 类的对象，`"abc".upper()` 就是在调用字符串对象的方法。

## 14.2 定义第一个类

```python
class Dog:
    def bark(self):
        print("汪汪汪！")

# 创建对象（实例化）
my_dog = Dog()        # 注意有括号
my_dog.bark()         # 汪汪汪！
```

- `class Dog:` 定义了一个名为 `Dog` 的类（类名习惯用**大驼峰**，即首字母大写，如 `MyClass`）。
- `bark` 是类里的**方法**（定义在类里的函数）。
- `Dog()` 创建了一个对象，`my_dog.bark()` 调用它的方法。

那个神秘的 `self` 是什么？往下看。

## 14.3 __init__ 与 self

大多数对象在创建时需要一些初始数据。`__init__` 是一个特殊方法（叫**构造方法/初始化方法**），在创建对象时**自动执行**，用来设置对象的初始属性：

```python
class Dog:
    def __init__(self, name, age):
        self.name = name      # 把参数保存为对象的属性
        self.age = age

    def bark(self):
        print(f"{self.name} 说：汪汪汪！")

# 创建对象时传入参数（会自动传给 __init__）
dog1 = Dog("旺财", 3)
dog2 = Dog("小黑", 5)

print(dog1.name)     # 旺财，访问属性
print(dog2.age)      # 5
dog1.bark()          # 旺财 说：汪汪汪！
dog2.bark()          # 小黑 说：汪汪汪！
```

### 理解 self（重点）

- `self` 代表**对象自己**。每个方法的第一个参数都必须是 `self`（名字是约定，可改但别改）。
- 当你写 `dog1.bark()` 时，Python 自动把 `dog1` 作为 `self` 传进去。所以方法内部通过 `self.name` 就能访问到**这个对象自己**的数据。
- `self.name = name` 的意思是：把传进来的参数 `name`，保存成“这个对象的 `name` 属性”。

> 初学者常问：为什么调用 `dog1.bark()` 不用传 self？因为 `dog1.` 这个“点”前面的对象会**自动**成为 self。你只需在定义时写上 self，调用时不用管它。

## 14.4 属性与方法

- **属性（attribute）**：对象拥有的数据，如 `dog1.name`、`dog1.age`。
- **方法（method）**：对象能执行的操作（定义在类里的函数），如 `dog1.bark()`。

方法内部可以自由使用和修改属性：

```python
class BankAccount:
    def __init__(self, owner, balance=0):
        self.owner = owner
        self.balance = balance

    def deposit(self, amount):
        self.balance += amount
        print(f"存入 {amount}，余额 {self.balance}")

    def withdraw(self, amount):
        if amount > self.balance:
            print("余额不足！")
        else:
            self.balance -= amount
            print(f"取出 {amount}，余额 {self.balance}")

acc = BankAccount("小明", 100)
acc.deposit(50)       # 存入 50，余额 150
acc.withdraw(200)     # 余额不足！
acc.withdraw(80)      # 取出 80，余额 70
```

这个例子很好地体现了 OOP 的思想：`balance`（数据）和 `deposit`/`withdraw`（操作）被封装在一起，外界通过方法来安全地操作数据。

## 14.5 实例属性 vs 类属性

- **实例属性**：在 `__init__` 里用 `self.xxx` 定义，**每个对象各有一份**。
- **类属性**：直接定义在类体中，**所有对象共享**。

```python
class Dog:
    species = "犬科"          # 类属性，所有狗共享

    def __init__(self, name):
        self.name = name       # 实例属性，每只狗不同

dog1 = Dog("旺财")
dog2 = Dog("小黑")

print(dog1.species)          # 犬科
print(dog2.species)          # 犬科（共享同一个）
print(Dog.species)           # 犬科（也能通过类名访问）

print(dog1.name, dog2.name)  # 旺财 小黑（各自不同）
```

类属性常用来存放“所有对象都一样的常量”或“统计信息”，比如统计一共创建了多少个对象：

```python
class Dog:
    count = 0                 # 记录总数

    def __init__(self, name):
        self.name = name
        Dog.count += 1        # 每创建一个就 +1

Dog("A"); Dog("B"); Dog("C")
print(Dog.count)             # 3
```

## 14.6 __str__：让对象更“可读”

直接打印一个对象，默认会显示一串没啥用的信息：

```python
class Dog:
    def __init__(self, name):
        self.name = name

d = Dog("旺财")
print(d)     # <__main__.Dog object at 0x000...>  看不懂
```

定义特殊方法 `__str__`，可以自定义打印时的显示内容：

```python
class Dog:
    def __init__(self, name, age):
        self.name = name
        self.age = age

    def __str__(self):
        return f"狗狗({self.name}, {self.age}岁)"

d = Dog("旺财", 3)
print(d)         # 狗狗(旺财, 3岁)
```

像 `__init__`、`__str__` 这种前后带双下划线的方法，称为**魔术方法/特殊方法**（dunder methods），Python 会在特定时机自动调用它们。第 15 章还会学到更多。

## 本章小结

- OOP 把**数据（属性）**和**操作（方法）**封装在一起。类是模板，对象是实例。
- `class 类名:` 定义类（大驼峰命名）；`类名()` 创建对象。
- `__init__(self, ...)` 是构造方法，创建对象时自动执行，用来初始化属性。
- `self` 代表对象自己；方法第一个参数必须是 self，调用时自动传入。
- 实例属性（`self.x`，各对象独立）vs 类属性（写在类体，所有对象共享）。
- `__str__` 自定义对象的打印显示。
