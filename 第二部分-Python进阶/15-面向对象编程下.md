## 15.1 继承：站在父类的肩膀上

现实中很多类有“共性 + 个性”。比如“猫”和“狗”都是“动物”，都有名字、都会吃东西（共性），但叫声不同（个性）。**继承**让我们把共性写在父类里，子类自动拥有，只需补充个性。

```python
class Animal:                # 父类（基类）
    def __init__(self, name):
        self.name = name

    def eat(self):
        print(f"{self.name} 在吃东西")

class Dog(Animal):           # 子类，括号里写父类名
    def bark(self):
        print(f"{self.name} 汪汪叫")

class Cat(Animal):
    def meow(self):
        print(f"{self.name} 喵喵叫")

d = Dog("旺财")
d.eat()      # 旺财 在吃东西（继承自 Animal）
d.bark()     # 旺财 汪汪叫（Dog 自己的）

c = Cat("咪咪")
c.eat()      # 咪咪 在吃东西
c.meow()     # 咪咪 喵喵叫
```

`Dog` 和 `Cat` 都继承了 `Animal`，自动获得了 `__init__` 和 `eat`，无需重复写。这就是继承带来的**代码复用**。

## 15.2 super()：调用父类方法

如果子类需要在父类的基础上**扩展** `__init__`，用 `super()` 先调用父类的初始化，再添加自己的属性：

```python
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)    # 调用父类的 __init__，设置 name
        self.breed = breed         # 再补充自己的属性

d = Dog("旺财", "金毛")
print(d.name)      # 旺财（父类设置的）
print(d.breed)     # 金毛（子类补充的）
```

`super()` 代表父类。`super().__init__(name)` 就是“执行父类的构造方法”。这样避免了重复写 `self.name = name`。

## 15.3 方法重写（Override）

子类可以重新定义（**重写**）从父类继承来的方法，实现不同的行为：

```python
class Animal:
    def speak(self):
        print("动物发出声音")

class Dog(Animal):
    def speak(self):                 # 重写父类的 speak
        print("汪汪汪")

class Cat(Animal):
    def speak(self):
        print("喵喵喵")

Animal().speak()    # 动物发出声音
Dog().speak()       # 汪汪汪（用子类自己的版本）
Cat().speak()       # 喵喵喵
```

有时重写时仍想保留父类的行为，可以在重写的方法里用 `super()` 调用父类版本，再追加内容。

## 15.4 多态：同样的调用，不同的表现

**多态（Polymorphism）**指的是：不同类的对象，面对同一个方法调用，各自表现不同。这让代码更灵活——我们可以用统一的方式处理不同类型的对象：

> 以下示例沿用 **§15.3** 中已定义的 `Animal`、`Dog`、`Cat` 类。注意：§15.3 的 `Animal` 是简化版（未定义 `__init__`），与 §15.1 中带 `name` 参数的版本不同，因此这里可以无参构造 `Dog()`。

```python
animals = [Dog(), Cat(), Animal()]
for animal in animals:
    animal.speak()      # 每个对象调用自己的 speak
# 汪汪汪
# 喵喵喵
# 动物发出声音
```

我们不需要关心 `animal` 具体是狗还是猫，只要它有 `speak()` 方法就行。这种“鸭子类型”（长得像鸭子、走起来像鸭子，就当它是鸭子）是 Python 的一大特色。

判断对象类型可以用 `isinstance()`：

```python
d = Dog()
print(isinstance(d, Dog))       # True
print(isinstance(d, Animal))    # True（Dog 也是 Animal）
```

## 15.5 封装与“私有”属性

**封装**的思想是：把对象内部的细节藏起来，只暴露必要的接口，防止外部乱改数据。

Python 没有严格的私有机制，而是用命名约定：

- `_name`：单下划线，表示“**这是内部使用的，请不要在外部直接访问**”（仅是约定，语法上仍可访问）。
- `__name`：双下划线，会触发“名称改写（name mangling）”，让外部更难直接访问，起到一定保护作用。

```python
class BankAccount:
    def __init__(self, balance):
        self.__balance = balance      # 双下划线，"私有"

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount

    def get_balance(self):
        return self.__balance

acc = BankAccount(100)
acc.deposit(50)
print(acc.get_balance())     # 150
# print(acc.__balance)       # ✗ 报错，外部访问不到
```

通过把 `__balance` 设为私有，外部只能通过 `deposit`（有校验）来改余额，不能随意 `acc.__balance = -9999`，从而保护了数据的合法性。

## 15.6 @property：把方法用成属性

有时我们想在“读取属性”时做一些计算或校验，`@property` 装饰器能把一个方法变得像属性一样访问（不用加括号）：

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius

    @property
    def area(self):                  # 像属性一样，不用 ()
        return 3.14159 * self.radius ** 2

c = Circle(5)
print(c.area)        # 78.53975  ← 注意没有括号！
```

还可以配合 `@属性名.setter` 在赋值时做校验：

```python
class Temperature:
    def __init__(self, celsius=0):
        self._celsius = celsius

    @property
    def celsius(self):
        return self._celsius

    @celsius.setter
    def celsius(self, value):
        if value < -273.15:
            raise ValueError("温度不能低于绝对零度")
        self._celsius = value

t = Temperature()
t.celsius = 25           # 触发 setter（含校验）
print(t.celsius)         # 25
# t.celsius = -300       # 触发 setter，抛出 ValueError
```

`@property` 让接口更优雅：外部像用普通属性一样使用，内部却能悄悄加逻辑。（装饰器本身的原理见第 16 章。）

## 15.7 类方法与静态方法（了解）

除了普通方法，还有两种特殊方法：

```python
class Pizza:
    count = 0

    def __init__(self, size):
        self.size = size
        Pizza.count += 1

    @classmethod
    def total_made(cls):             # 类方法，第一个参数是 cls（类本身）
        return cls.count

    @staticmethod
    def is_valid_size(size):         # 静态方法，不需要 self 或 cls
        return size in ("小", "中", "大")

Pizza("大"); Pizza("中")
print(Pizza.total_made())            # 2
print(Pizza.is_valid_size("超大"))   # False
```

- `@classmethod`：操作的是类本身（`cls`），常用于统计、或作为“工厂方法”创建对象。
- `@staticmethod`：只是逻辑上属于这个类的工具函数，既不用对象也不用类的数据。

初学阶段知道有这两种即可，普通方法（带 self 的）是最常用的。

## 本章小结

- **继承**：`class 子类(父类):`，子类自动获得父类的属性和方法，实现复用。
- `super().__init__(...)` 调用父类构造方法；子类可**重写**父类方法。
- **多态**：不同对象对同一方法调用有不同表现；`isinstance()` 判断类型。
- **封装**：`_x`（约定内部用）、`__x`（更强的私有），通过方法安全地操作数据。
- `@property` 把方法当属性访问，可配 `setter` 做校验。
- `@classmethod`（操作类，参数 cls）与 `@staticmethod`（工具函数）。

## 练习题

1. **继承体系**：定义父类 `Shape`（含方法 `area()` 返回 0），子类 `Square`（正方形）和 `Circle`（圆）分别重写 `area()`。用一个列表装几个图形，循环打印各自面积（体会多态）。
2. **员工与经理**：`Employee` 有 `name`、`salary`，方法 `info()`。`Manager` 继承它，额外有 `bonus`，重写 `info()` 显示含奖金的总收入（用 `super()`）。
3. **私有余额**：改造银行账户类，把余额设为私有 `__balance`，只能通过 `deposit`/`withdraw` 修改，`withdraw` 要防止透支。
4. **温度属性**：写一个 `Temperature` 类，用 `@property` 提供一个只读属性 `fahrenheit`，根据摄氏度自动换算成华氏度。
5. **图书统计**：`Book` 类每创建一本就让类属性 `total` 加一，用 `@classmethod` 提供 `count()` 返回总数。
