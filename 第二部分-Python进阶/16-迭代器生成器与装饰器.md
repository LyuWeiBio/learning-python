## 16.1 可迭代对象与迭代器

我们能用 `for` 遍历的对象（列表、字符串、字典、集合、range……）都叫**可迭代对象（iterable）**。

`for` 循环的背后，其实是这样工作的：先用 `iter()` 拿到一个**迭代器（iterator）**，然后不断调用 `next()` 取下一个值，直到没有值时抛出 `StopIteration` 停止。

```python
nums = [10, 20, 30]
it = iter(nums)          # 得到迭代器
print(next(it))          # 10
print(next(it))          # 20
print(next(it))          # 30
# print(next(it))        # StopIteration（没有更多元素）
```

平时我们用 `for` 就够了，不用手动 `next()`。理解这个机制，是为了看懂下面的生成器。

## 16.2 生成器：用 yield 惰性产生数据

**生成器**是一种特殊的、写起来极其简单的迭代器。把函数里的 `return` 换成 `yield`，它就变成了生成器函数：

```python
def count_up_to(n):
    i = 1
    while i <= n:
        yield i          # 每次 yield 一个值，然后"暂停"
        i += 1

gen = count_up_to(5)
for num in gen:
    print(num)           # 1 2 3 4 5
```

**关键区别**：普通函数 `return` 一次就结束；生成器每次 `yield` 会“吐出”一个值并**暂停**，下次取值时从暂停处继续。

### 惰性求值：省内存的秘诀

生成器**不会一次性生成所有数据**，而是“要一个给一个”。这对处理大量数据非常有用——不必把所有数据都塞进内存。

```python
# 列表：立刻在内存中生成 1000 万个数，很占内存
big_list = [x for x in range(10_000_000)]

# 生成器：几乎不占内存，用到才算
big_gen = (x for x in range(10_000_000))   # 圆括号 = 生成器表达式
```

**生成器表达式**和列表推导式几乎一样，只是把 `[]` 换成 `()`：

```python
gen = (x ** 2 for x in range(5))
print(sum(gen))          # 30，边生成边求和，不占额外内存
```

一个实用场景——逐行读取超大文件并处理，内存占用极低：

```python
def read_large_file(path):
    with open(path, encoding="utf-8") as f:
        for line in f:
            yield line.strip()

# for line in read_large_file("huge.txt"):
#     process(line)
```

## 16.3 装饰器：给函数“增强”功能

**装饰器（decorator）**能在**不修改原函数代码**的前提下，给它增加额外功能（比如计时、记录日志、权限检查）。这在框架中极为常见（比如 Web 框架的路由、第 15 章的 `@property`）。

### 前置知识：函数是“一等公民”

在 Python 里，函数和普通值一样，可以被赋值给变量、当参数传递、被别的函数返回：

```python
def shout(text):
    return text.upper()

yell = shout             # 把函数赋给变量（注意没有括号）
print(yell("hello"))     # HELLO

def apply(func, value):  # 函数当参数
    return func(value)
print(apply(shout, "hi"))   # HI
```

### 手写一个装饰器

装饰器本质是“一个接收函数、返回新函数的函数”：

```python
def timer(func):
    """装饰器：打印函数运行耗时。"""
    import time
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)      # 调用原函数
        end = time.time()
        print(f"{func.__name__} 耗时 {end - start:.4f} 秒")
        return result
    return wrapper       # 返回增强后的新函数

def slow_add(a, b):
    import time
    time.sleep(0.5)
    return a + b

slow_add = timer(slow_add)   # 用装饰器"包裹"原函数
print(slow_add(3, 5))
# slow_add 耗时 0.50xx 秒
# 8
```

### @ 语法糖

上面 `slow_add = timer(slow_add)` 这一步，Python 提供了更漂亮的写法——在函数定义前加 `@装饰器名`：

```python
@timer                   # 等价于 slow_add = timer(slow_add)
def slow_add(a, b):
    import time
    time.sleep(0.5)
    return a + b

print(slow_add(3, 5))    # 自动计时
```

这就是你在各种代码里看到的 `@xxx` 的来历——它就是装饰器。

### 保留原函数信息：functools.wraps

装饰后，函数的名字、文档会变成 `wrapper` 的，这会造成困扰。用 `functools.wraps` 可以保留原函数信息：

```python
import functools

def timer(func):
    @functools.wraps(func)       # 保留原函数的 __name__、__doc__
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

写装饰器时加上 `@functools.wraps(func)` 是一个好习惯。

## 16.4 一个更实用的装饰器例子

带“重试”功能的装饰器：函数出错时自动重试几次。

```python
import functools

def retry(times=3):
    """出错时最多重试 times 次的装饰器。"""
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):
            for i in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as e:
                    print(f"第 {i + 1} 次失败：{e}")
            print("重试次数用尽")
        return wrapper
    return decorator

@retry(times=3)
def risky():
    import random
    if random.random() < 0.7:
        raise ValueError("随机失败")
    return "成功！"

print(risky())
```

这种“带参数的装饰器”多了一层嵌套，理解起来稍难，作为拓展了解即可。

## 本章小结

- 可迭代对象能被 `for` 遍历；底层通过 `iter()`/`next()` 取值，耗尽时 `StopIteration`。
- **生成器**用 `yield` 定义，每次产出一个值并暂停，**惰性求值、省内存**。
- 生成器表达式 `(x for x in ...)`：像列表推导式但用圆括号。
- **装饰器**在不改原函数的前提下增强功能；本质是“接收函数、返回新函数”。
- `@装饰器名` 是语法糖，等价于 `f = 装饰器(f)`。
- 写装饰器时用 `@functools.wraps(func)` 保留原函数信息。

5. **理解题**：解释为什么下面代码打印的是 `wrapper` 而不是 `hello`，以及如何修复。

```python
def deco(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper

@deco
def hello():
    """打个招呼"""
    print("hi")

print(hello.__name__)     # 打印出的是 wrapper
```

```python
import functools

def log_call(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print(f"调用 {func.__name__}")
        result = func(*args, **kwargs)
        print(f"{func.__name__} 执行完毕")
        return result
    return wrapper

@log_call
def add(a, b):
    return a + b

print(add(3, 5))
```

**第 4 题**：

```python
import functools

def count_calls(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        wrapper.calls += 1
        print(f"第 {wrapper.calls} 次调用")
        return func(*args, **kwargs)
    wrapper.calls = 0
    return wrapper

@count_calls
def greet():
    print("你好")

greet(); greet()      # 第 1 次调用 / 第 2 次调用
```

**第 5 题**：因为装饰后 `hello` 实际指向的是内部的 `wrapper` 函数，所以 `__name__` 是 `"wrapper"`。修复方法是给 `wrapper` 加上 `@functools.wraps(func)`：

```python
import functools
def deco(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```
