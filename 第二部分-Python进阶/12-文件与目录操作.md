## 12.1 为什么要操作文件

程序运行结束后，内存里的数据就消失了。要想**持久保存**数据（比如把结果写进文件、下次读回来），或者**读取外部数据**（比如一份成绩单、一个数据集），就需要文件操作。这也是通往数据分析的必经之路。

---

## 12.2 打开与关闭文件

基本流程：打开 → 读/写 → 关闭。

```python
f = open("test.txt", "w", encoding="utf-8")   # 打开
f.write("你好，文件")                            # 写入
f.close()                                       # 关闭（很重要！）
```

`open()` 的常用参数：

- 第一个参数：**文件路径**。
- 第二个参数：**模式**。
- `encoding`：**编码**，处理中文强烈建议写 `"utf-8"`，否则在某些系统上会乱码或报错。

### 文件打开模式

| 模式 | 含义 | 文件不存在时 | 文件已存在时 |
| --- | --- | --- | --- |
| `"r"` | 只读（默认） | 报错 | 从头读 |
| `"w"` | 写入 | 新建 | **清空**后写入 |
| `"a"` | 追加 | 新建 | 在末尾追加 |
| `"x"` | 独占创建 | 新建 | 报错 |
| `"r+"` | 读写 | 报错 | 可读可写 |

> ⚠️ **`"w"` 模式会清空原文件内容**！如果只是想在文件末尾添加内容，一定要用 `"a"`（追加）模式。

---

## 12.3 with 语句（推荐做法）

手动 `close()` 容易忘记，一旦中途报错还可能导致文件没正常关闭。**`with` 语句**能自动帮我们关闭文件，是官方推荐的写法：

```python
with open("test.txt", "w", encoding="utf-8") as f:
    f.write("第一行\n")
    f.write("第二行\n")
# 缩进块结束，文件自动关闭，无需手动 close()
```

从今往后，**请一律使用 `with` 来操作文件**。它属于“上下文管理器”，能确保资源被正确释放。

---

## 12.4 写入文件

```python
lines = ["苹果", "香蕉", "橙子"]

with open("fruits.txt", "w", encoding="utf-8") as f:
    for fruit in lines:
        f.write(fruit + "\n")       # write 不会自动加换行，要手动加 \n

# 或者用 writelines 一次写入多行
with open("fruits.txt", "w", encoding="utf-8") as f:
    f.writelines(fruit + "\n" for fruit in lines)
```

**追加**内容（不清空原文件）：

```python
with open("fruits.txt", "a", encoding="utf-8") as f:
    f.write("葡萄\n")
```

## 12.5 读取文件

### 一次性读取全部

```python
with open("fruits.txt", "r", encoding="utf-8") as f:
    content = f.read()        # 返回整个文件的字符串
    print(content)
```

### 逐行读取（推荐处理大文件）

直接对文件对象用 `for` 循环，会一行一行地读，非常省内存：

```python
with open("fruits.txt", "r", encoding="utf-8") as f:
    for line in f:
        print(line.strip())    # strip() 去掉每行末尾的换行符
```

### 读成列表

```python
with open("fruits.txt", "r", encoding="utf-8") as f:
    lines = f.readlines()      # 每行作为一个元素，含换行符
    print(lines)               # ['苹果\n', '香蕉\n', ...]
```

> **注意**：`read()` 后再 `read()` 会得到空字符串，因为“读取位置”已经到文件末尾了。文件像一个磁头，读过就往后走。

## 12.6 用 pathlib 处理路径

现代 Python 推荐用 `pathlib` 模块处理文件路径，它比字符串拼接更安全、跨平台：

```python
from pathlib import Path

p = Path("data") / "fruits.txt"   # 用 / 拼接路径，跨平台
print(p)                           # data/fruits.txt

print(p.exists())                  # 文件是否存在
print(p.name)                      # fruits.txt（文件名）
print(p.suffix)                    # .txt（扩展名）
print(p.parent)                    # data（父目录）

# 创建目录（parents=True 自动创建多级，exist_ok=True 已存在也不报错）
Path("output").mkdir(parents=True, exist_ok=True)

# pathlib 还能直接读写小文件
Path("hello.txt").write_text("你好", encoding="utf-8")
print(Path("hello.txt").read_text(encoding="utf-8"))
```

判断文件是否存在，再决定读还是写，是很常见的模式：

```python
from pathlib import Path

path = Path("config.txt")
if path.exists():
    print(path.read_text(encoding="utf-8"))
else:
    print("配置文件不存在，使用默认设置")
```

## 12.7 读写 CSV 文件

CSV（逗号分隔值）是最常见的表格数据格式，Excel 也能打开。标准库 `csv` 模块可以处理它：

```python
import csv

# 写入 CSV
data = [
    ["姓名", "年龄", "城市"],
    ["小明", 18, "北京"],
    ["小红", 20, "上海"],
]
with open("people.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.writer(f)
    writer.writerows(data)      # 一次写入多行

# 读取 CSV
with open("people.csv", "r", encoding="utf-8") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)              # 每行是一个列表
```

> 写 CSV 时加 `newline=""` 参数，能避免在 Windows 上出现多余空行。到了第三部分，我们会用更强大的 **Pandas** 来读写 CSV，但理解底层原理很有帮助。

## 12.8 综合示例：日志记录器

```python
from datetime import datetime

def log(message, filename="app.log"):
    """把带时间戳的日志追加到文件。"""
    timestamp = datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    with open(filename, "a", encoding="utf-8") as f:
        f.write(f"[{timestamp}] {message}\n")

log("程序启动")
log("用户登录")
log("程序结束")

# 读出来看看
with open("app.log", "r", encoding="utf-8") as f:
    print(f.read())
```

## 本章小结

- `open(路径, 模式, encoding="utf-8")`；处理中文务必指定 utf-8 编码。
- 模式：`r` 读、`w` 写（**会清空**）、`a` 追加。
- **一律用 `with open(...) as f:`**，自动关闭文件、更安全。
- 读取：`read()` 全部、`for line in f` 逐行、`readlines()` 成列表；`strip()` 去换行。
- 写入：`write()` 不自动换行，需手动加 `\n`。
- `pathlib.Path` 处理路径：`/` 拼接、`exists()`、`mkdir()`、`read_text/write_text`。
- `csv` 模块读写表格数据。
