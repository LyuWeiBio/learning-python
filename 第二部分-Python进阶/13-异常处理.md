# 第 13 章：异常处理

> **所属部分**：第二部分 · Python 进阶
> **预计学习时间**：45 分钟

## 本章学习目标

- 学会阅读报错信息（Traceback）。
- 掌握 `try / except` 捕获并处理异常。
- 会区分不同异常类型，会获取异常对象。
- 掌握 `else` 和 `finally` 子句。
- 会用 `raise` 主动抛出异常。

---

## 13.1 错误与异常

程序运行中出现的问题分两类：

- **语法错误（SyntaxError）**：代码写错了，程序根本无法运行，比如少写冒号、括号不匹配。这类错误必须先改对。
- **异常（Exception）**：语法没问题，但**运行时**出了岔子，比如除以零、把字符串转成整数失败、访问不存在的文件。

异常如果不处理，程序会**崩溃并终止**。异常处理就是让程序在遇到问题时能“优雅地应对”，而不是直接崩掉。

---

## 13.2 读懂 Traceback

先看一个会崩溃的例子：

```python
num = int("abc")     # "abc" 无法转成整数
```

运行会得到类似输出：

```
Traceback (most recent call last):
  File "test.py", line 1, in <module>
    num = int("abc")
ValueError: invalid literal for int() with base 10: 'abc'
```

**阅读技巧**：

- 从**最后一行**看起，它告诉你**异常类型**（`ValueError`）和**原因**（无法转换 `'abc'`）。
- 中间部分显示出错的**文件、行号、代码**，帮你定位。
- 如果调用层级很深，Traceback 会从上到下列出调用链，**最下面的才是真正出错的地方**。

> 遇到报错先别慌，认真读最后一行，90% 的问题都能自己解决。

### 常见异常类型

| 异常 | 触发场景 |
| --- | --- |
| `ValueError` | 值不合适，如 `int("abc")` |
| `TypeError` | 类型不对，如 `"a" + 1` |
| `ZeroDivisionError` | 除以零 |
| `IndexError` | 列表索引越界 |
| `KeyError` | 字典键不存在 |
| `FileNotFoundError` | 文件找不到 |
| `NameError` | 使用了未定义的变量 |
| `AttributeError` | 对象没有该属性/方法 |

## 13.3 try / except：捕获异常

把可能出错的代码放进 `try` 块，把出错后的处理放进 `except` 块：

```python
try:
    num = int(input("请输入一个整数："))
    print(f"你输入的是 {num}")
except ValueError:
    print("输入的不是有效整数！")
```

如果用户输入 `abc`，程序不会崩溃，而是打印友好的提示。

执行逻辑：`try` 块正常执行完就跳过 `except`；一旦 `try` 中某行抛出异常，立即跳到匹配的 `except` 块。

---

## 13.4 捕获多种异常

一个 `try` 可以配多个 `except`，分别处理不同异常：

```python
try:
    a = int(input("被除数："))
    b = int(input("除数："))
    print(a / b)
except ValueError:
    print("请输入整数！")
except ZeroDivisionError:
    print("除数不能为 0！")
```

也可以用一个元组同时捕获多种：

```python
try:
    ...
except (ValueError, TypeError):
    print("值或类型有误")
```

### 获取异常对象

用 `as` 拿到异常对象，可以查看具体信息：

```python
try:
    num = int("abc")
except ValueError as e:
    print(f"出错了：{e}")      # 出错了：invalid literal for int()...
    print(f"异常类型：{type(e).__name__}")   # ValueError
```

> **注意**：避免直接写 `except:`（裸捕获）或 `except Exception:` 却什么都不做。这会把所有错误（包括你没想到的 bug）都吞掉，让问题更难发现。**尽量捕获具体的异常类型。**

---

## 13.5 else 与 finally

```python
try:
    num = int(input("请输入："))
except ValueError:
    print("输入无效")
else:
    print(f"输入成功：{num}")     # try 没出错时才执行
finally:
    print("无论如何都会执行")      # 常用于清理资源
```

- `else`：`try` 块**没有发生异常**时执行。
- `finally`：**无论是否异常都会执行**，常用于释放资源（关闭文件、断开连接等）。

---

## 13.6 raise：主动抛出异常

有时我们希望在检测到非法情况时，**主动**抛出异常来中断程序，让调用者知道出错了：

```python
def set_age(age):
    if age < 0:
        raise ValueError("年龄不能为负数")
    if age > 150:
        raise ValueError("年龄不合理")
    print(f"年龄设置为 {age}")

set_age(25)          # 正常
set_age(-5)          # 抛出 ValueError: 年龄不能为负数
```

`raise` 后面跟一个异常类型和描述信息。这在编写函数、校验参数时很有用——**与其让错误数据继续流转、在更深处引发莫名其妙的问题，不如尽早抛出清晰的异常。**

## 13.7 自定义异常（了解）

可以定义自己的异常类型（继承自 `Exception`，类的知识见第 14 章）：

```python
class InsufficientBalanceError(Exception):
    """余额不足时抛出。"""
    pass

def withdraw(balance, amount):
    if amount > balance:
        raise InsufficientBalanceError(f"余额 {balance} 不足以支取 {amount}")
    return balance - amount

try:
    withdraw(100, 200)
except InsufficientBalanceError as e:
    print(f"取款失败：{e}")
```

自定义异常能让错误类型更有业务含义，大型项目里很常用。

## 13.8 实战：健壮的输入函数

综合运用，写一个“不断重试直到用户输入合法整数”的函数：

```python
def input_int(prompt):
    while True:
        try:
            return int(input(prompt))
        except ValueError:
            print("请输入一个整数，再试一次。")

age = input_int("请输入年龄：")
print(f"你的年龄是 {age}")
```

这个模式把 `try/except`、`while True`、`return` 结合在一起，是非常实用的“防呆”写法。

## 本章小结

- 语法错误让程序无法运行；异常是运行时问题，不处理会崩溃。
- 读 Traceback 先看**最后一行**（异常类型 + 原因）。
- `try/except` 捕获异常；可配多个 `except`，用 `as e` 获取异常对象。
- 尽量捕获**具体**异常，别用裸 `except` 把错误全吞掉。
- `else`（无异常时执行）、`finally`（总会执行，用于清理）。
- `raise` 主动抛异常，用于参数校验；可自定义异常类型。

## 练习题

1. **安全除法**：写函数 `safe_divide(a, b)`，正常返回 `a/b`，除数为 0 时捕获异常并返回 `None`。
2. **列表取值**：写函数 `get_item(lst, index)`，返回 `lst[index]`；索引越界时捕获 `IndexError` 并返回字符串 `"越界"`。
3. **读文件**：写函数 `read_file(path)`，尝试读取文件内容返回；若文件不存在，捕获 `FileNotFoundError` 并返回 `"文件不存在"`。
4. **校验分数**：写函数 `set_score(score)`，若分数不在 0~100 之间就 `raise ValueError`，否则打印分数。用 `try/except` 测试传入 150 的情况。
5. **健壮的计算器**：让用户输入两个数和一个运算符（+ - * /），用异常处理应对“非数字输入”和“除以零”，给出友好提示。
